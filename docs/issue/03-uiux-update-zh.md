### 改进虚拟键盘
- 当前问题：目前的键盘布局是固定的 QWERTY 布局。
- 建议：
    - 数字键盘优化：在输入 serverUrl 中的端口号或 IP 地址时，应该自动切换为“九宫格数字键盘”或“纯数字行”，现在的键盘按数字键需要切换层级，效率较低。
    - 遮挡处理：当前键盘是 fixed bottom-0。如果在编辑屏幕底部的输入框，键盘弹出可能会遮挡住输入框本身。建议引入 scrollIntoView 逻辑，键盘弹出时自动将输入框顶上去。

### 硬件调试页面的终端体验
- 当前问题：HardwarePage.vue 中的终端日志区域高度固定 (h-56 / h-72)。
- 建议：
    - 自动滚动：当新日志产生时，应该自动滚动到底部（scrollTop = scrollHeight），目前代码中似乎没有看到这个逻辑，维护人员可能需要手动去划屏幕看最新结果。
    - 等宽字体：虽然定义了 font-mono，但建议在深色背景下稍微增加字间距 (tracking-wide)，提高在低分辨率工控屏上的可读性。

### 动画与性能平衡
- 当前问题：global.css 中定义了 ripple (波纹) 和 blur (毛玻璃) 效果。
- 风险：很多工控机的 GPU 性能极差（通常是低端集成显卡）。backdrop-filter: blur() 在低端设备上会导致严重的掉帧或卡顿。
- 建议：通过媒体查询检测设备性能，在低端设备上用纯色半透明背景代替毛玻璃效果。

### 报错信息的友好度
- 当前问题：device.ts 和部分页面中的 catch 块处理比较简单，直接 Toast 提示“操作失败”。
- 建议：对于 Kiosk，错误代码至关重要。建议在 Toast 中显示错误代码（如 Error: E02），方便现场人员拍照发给开发人员定位问题，而不是只显示模糊的“连接失败”。
